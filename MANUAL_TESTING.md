# Руководство по ручному тестированию

Данное руководство содержит пошаговые инструкции для ручного тестирования всех функций бота.

## Подготовка к тестированию

### 1. Настройка окружения
- [ ] Убедитесь, что PostgreSQL запущен
- [ ] Выполните миграции: `npm run migrate`
- [ ] Настройте переменные окружения в `.env`
- [ ] Запустите бота: `npm run dev`

### 2. Создание тестового пользователя
- [ ] Найдите бота в Telegram
- [ ] Отправьте команду `/start`
- [ ] Убедитесь, что получили приветственное сообщение

## 9.1 Тестирование основного флоу

### Тест 1: Фото без подписи
- [ ] Отправьте фото еды без подписи
- [ ] Ожидаемый результат:
  - [ ] Бот отправляет сообщение "⏳ Анализирую фото..."
  - [ ] Через некоторое время приходит результат с:
    - [ ] 🍽️ Название блюда
    - [ ] 🔥 Калории (ккал)
    - [ ] 🥩 Белки (г)
    - [ ] 🧈 Жиры (г)
    - [ ] 🍞 Углеводы (г)
  - [ ] Есть кнопка "Корректировать результаты"

### Тест 2: Фото с весом в подписи
- [ ] Отправьте фото с подписью "250г"
- [ ] Ожидаемый результат:
  - [ ] Результат учитывает указанный вес
  - [ ] Пищевая ценность рассчитана для 250г

### Тест 3: Фото с текстом и весом
- [ ] Отправьте фото с подписью "Мой обед 150г"
- [ ] Ожидаемый результат:
  - [ ] Бот извлекает вес 150г
  - [ ] Игнорирует остальной текст

### Тест 4: Проверка форматирования
- [ ] Убедитесь, что числа округлены до 1 знака после запятой
- [ ] Проверьте наличие всех эмодзи
- [ ] Убедитесь в читаемости сообщения

**Статус теста 9.1:** ✅ Пройден / ❌ Не пройден

## 9.2 Тестирование системы лимитов

### Тест 1: Первый запрос FREE пользователя
- [ ] Новый пользователь отправляет первое фото
- [ ] Ожидаемый результат:
  - [ ] Запрос обрабатывается успешно
  - [ ] Получен результат анализа

### Тест 2: Исчерпание дневного лимита
- [ ] Отправьте второе фото (лимит FREE = 1)
- [ ] Ожидаемый результат:
  - [ ] Сообщение о достижении лимита
  - [ ] Кнопки для покупки запросов:
    - [ ] "10 запросов за 99 ₽"
    - [ ] "50 запросов за 399 ₽"
    - [ ] "100 запросов за 699 ₽"

### Тест 3: Использование купленных запросов
- [ ] Добавьте пользователю purchased_requests в БД
- [ ] Отправьте фото при исчерпанном дневном лимите
- [ ] Ожидаемый результат:
  - [ ] Запрос обрабатывается
  - [ ] purchased_requests уменьшается на 1

### Тест 4: PREMIUM пользователь
- [ ] Измените user_class_id на 2 (PREMIUM) в БД
- [ ] Отправьте много фото (>10)
- [ ] Ожидаемый результат:
  - [ ] Все запросы обрабатываются
  - [ ] Нет ограничений

### Тест 5: Команда /status
- [ ] Отправьте `/status`
- [ ] Ожидаемый результат для FREE:
  - [ ] Класс: FREE
  - [ ] Использовано сегодня: X/1
  - [ ] Купленных запросов: X
- [ ] Ожидаемый результат для PREMIUM:
  - [ ] Класс: PREMIUM
  - [ ] Безлимитный доступ

**Статус теста 9.2:** ✅ Пройден / ❌ Не пройден

## 9.3 Тестирование корректировки результатов

### Тест 1: Открытие меню корректировки
- [ ] Отправьте фото и получите результат
- [ ] Нажмите "Корректировать результаты"
- [ ] Ожидаемый результат:
  - [ ] Появляются кнопки:
    - [ ] 🔥 Калории
    - [ ] 🥩 Белки
    - [ ] 🧈 Жиры
    - [ ] 🍞 Углеводы
    - [ ] ❌ Отмена

### Тест 2: Корректировка калорий
- [ ] Нажмите "🔥 Калории"
- [ ] Введите "400"
- [ ] Ожидаемый результат:
  - [ ] Сообщение "✅ Калории обновлены"
  - [ ] Исходное сообщение обновлено с новым значением

### Тест 3: Корректировка каждого параметра
Повторите для каждого параметра:
- [ ] Белки: введите "30"
- [ ] Жиры: введите "20"
- [ ] Углеводы: введите "45"

### Тест 4: Валидация ввода
Проверьте отклонение некорректных значений:
- [ ] Отрицательное число "-10" → ❌ Ошибка
- [ ] Ноль "0" → ❌ Ошибка
- [ ] Текст "abc" → ❌ Ошибка
- [ ] Пустая строка → ❌ Ошибка
- [ ] Десятичное число "25.5" → ✅ Принято

### Тест 5: Отмена корректировки
- [ ] Нажмите "❌ Отмена"
- [ ] Ожидаемый результат:
  - [ ] Возврат к кнопке "Корректировать результаты"
  - [ ] Сообщение "Отменено"

### Тест 6: Проверка сохранения в БД
- [ ] Выполните корректировку
- [ ] Проверьте в БД таблицу requests_history
- [ ] Убедитесь, что значения обновлены

**Статус теста 9.3:** ✅ Пройден / ❌ Не пройден

## 9.4 Тестирование обработки ошибок

### Тест 1: Недоступность OpenAI API
- [ ] Временно измените OPENAI_API_KEY на неверный
- [ ] Отправьте фото
- [ ] Ожидаемый результат:
  - [ ] Сообщение "⚠️ Сервис временно недоступен. Попробуйте позже."
  - [ ] В логах есть запись об ошибке

### Тест 2: Ошибка базы данных
- [ ] Остановите PostgreSQL
- [ ] Отправьте команду `/status`
- [ ] Ожидаемый результат:
  - [ ] Сообщение "⚠️ Произошла ошибка. Попробуйте позже."
  - [ ] В логах есть запись об ошибке подключения

### Тест 3: Некорректное фото
- [ ] Отправьте очень маленькое или поврежденное изображение
- [ ] Ожидаемый результат:
  - [ ] Понятное сообщение об ошибке
  - [ ] Бот не падает

### Тест 4: Сетевые проблемы
- [ ] Временно отключите интернет
- [ ] Отправьте фото
- [ ] Ожидаемый результат:
  - [ ] Сообщение о недоступности сервиса
  - [ ] Бот продолжает работать после восстановления связи

### Тест 5: Проверка логирования
- [ ] Проверьте файлы логов (error.log, combined.log)
- [ ] Убедитесь, что ошибки записываются с:
  - [ ] Временной меткой
  - [ ] ID пользователя
  - [ ] Описанием ошибки
  - [ ] Stack trace (для критических ошибок)

### Тест 6: Graceful degradation
- [ ] Создайте ситуацию частичного сбоя (например, OpenAI работает, БД нет)
- [ ] Убедитесь, что бот:
  - [ ] Не падает полностью
  - [ ] Продолжает отвечать на команды
  - [ ] Восстанавливается после устранения проблемы

**Статус теста 9.4:** ✅ Пройден / ❌ Не пройден

## Итоговая проверка

### Общие требования
- [ ] Все тесты пройдены успешно
- [ ] Бот стабильно работает под нагрузкой
- [ ] Логи ведутся корректно
- [ ] Ошибки обрабатываются gracefully
- [ ] Пользователи получают понятные сообщения

### Производительность
- [ ] Время ответа на фото < 30 секунд
- [ ] Команды отвечают мгновенно
- [ ] База данных не перегружается

### Безопасность
- [ ] API ключи не попадают в логи
- [ ] Внутренние ошибки не раскрываются пользователям
- [ ] Валидация всех пользовательских вводов
