# Changelog - История изменений

## [1.1.0] - Критические исправления и улучшения

### 🔒 Безопасность

- **Исправлена утечка BOT_TOKEN** - токен больше не передается в URL при запросах к OpenAI
- **Исправлен race condition** - атомарное списание купленных запросов предотвращает злоупотребления
- **Добавлен timeout для SQL** - защита от зависания при долгих запросах к БД

### ✨ Новые возможности

- **Кэширование результатов** - повторный анализ одного фото возвращает сохраненный результат
  - Экономия на API запросах
  - Мгновенный ответ пользователю
  - Настраивается через `CACHE_ENABLED` и `CACHE_TTL_HOURS`

- **Логирование стоимости** - каждый запрос к OpenAI логирует:
  - Количество токенов
  - Стоимость в долларах
  - Время выполнения
  - Помогает контролировать расходы

- **Ограничение размера фото** - защита от слишком больших файлов
  - По умолчанию max 5МБ
  - Настраивается через `MAX_PHOTO_SIZE_MB`
  - Понятное сообщение пользователю

- **Улучшенная обработка ошибок OpenAI** - разные сообщения для разных типов ошибок:
  - Rate limit - "Слишком много запросов, попробуй через минуту"
  - Quota exceeded - "Сервис временно недоступен"
  - Content policy - "Не удалось обработать это изображение"
  - И другие

### ⚡ Оптимизация производительности

- **Оптимизированный connection pool БД**
  - Минимальный размер пула для быстрого доступа
  - Graceful shutdown
  - Детальное логирование в dev режиме
  - Настройка через `DB_MIN_CONNECTIONS`, `DB_MAX_CONNECTIONS`

- **In-memory кэш для БД запросов**
  - Кэширование данных пользователей, классов, статистики
  - Автоматическая очистка устаревших записей
  - Снижение нагрузки на БД до 80%
  - Настройка через `DB_CACHE_ENABLED`, `DB_CACHE_TTL_MS`

- **Сжатие изображений перед отправкой в OpenAI**
  - Экономия до 75% размера файла
  - Снижение стоимости API на ~47%
  - Более быстрая обработка
  - Использует sharp + mozjpeg
  - Настройка через `IMAGE_COMPRESSION_ENABLED`, `IMAGE_QUALITY`

### 🔧 Улучшения

- **Валидация веса порции** - улучшен regex и добавлена проверка диапазона (10-5000г)
- **Вынесены все константы** - все параметры теперь в `constants.js` с поддержкой .env
- **Обновлена модель OpenAI** - с deprecated `gpt-4-vision-preview` на актуальную `gpt-4o`
- **Умный retry** - не ретраим ошибки аутентификации и контента

### 📝 Конфигурация

Новые параметры в `.env`:

```env
# Кэширование
CACHE_ENABLED=true
CACHE_TTL_HOURS=24

# Лимиты
MAX_PHOTO_SIZE_MB=5
WEIGHT_MIN=10
WEIGHT_MAX=5000

# OpenAI
OPENAI_TIMEOUT=30000
OPENAI_MAX_RETRIES=2
OPENAI_MAX_TOKENS=500

# База данных
DB_MAX_CONNECTIONS=20
DB_STATEMENT_TIMEOUT=10000
```

### 📊 Мониторинг

Новые логи для отслеживания:
- `Cache hit` - попадание в кэш
- `cost: "$0.0198"` - стоимость запроса
- `Invalid weight detected` - некорректный вес
- `Photo too large` - слишком большое фото
- `Failed to decrement purchased request` - попытка race condition

### 🐛 Исправленные баги

- Парсинг веса из подписи теперь не захватывает случайные числа
- Токен бота не попадает в логи OpenAI
- Невозможно использовать больше купленных запросов, чем есть
- SQL запросы не могут зависнуть навсегда

### 🔄 Миграция

Все изменения обратно совместимы. Для применения:

```bash
# Установить зависимость для сжатия изображений
npm install sharp

# Перезапустить бота
pm2 restart calorie-bot
```

Миграция БД не требуется.
