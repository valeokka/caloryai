# Изменения в OpenAI сервисе

## Что изменилось

### 1. Модель
- Жёстко задана `gpt-4o-mini` (вместо переменной из конфига)
- Убраны лишние константы из .env

### 2. Качество изображения
- Используется `detail: 'low'` (85 токенов фиксированно)
- Экономия ~70% токенов на изображениях

### 3. Промпт
Новый ультракороткий промпт:
```
Analyze food. Weight: 250g. Return JSON: {"name":"","weight":250,"kcal":0,"protein":0,"fat":0,"carbs":0}
```

Или без веса:
```
Analyze food. Estimate weight. Return JSON: {"name":"","weight":0,"kcal":0,"protein":0,"fat":0,"carbs":0}
```

### 4. Формат ответа от OpenAI
Теперь получаем:
```json
{
  "name": "Куриная грудка с овощами",
  "weight": 250,
  "kcal": 350,
  "protein": 25,
  "fat": 15,
  "carbs": 30
}
```

### 5. Внутренняя обработка
- Парсим JSON от OpenAI
- Валидируем новую структуру (name, weight, kcal, protein, fat, carbs)
- Преобразуем в старый формат для совместимости с остальным кодом

### 6. Оптимизация
- `max_tokens: 150` (вместо 100)
- `temperature: 0.2` (вместо 0.3) - меньше креативности
- Убраны лишние комментарии
- Упрощена обработка ошибок

## Совместимость
Код полностью совместим с существующими обработчиками и тестами, т.к. на выходе возвращается тот же формат:
```javascript
{
  dishName: "...",
  calories: 350,
  protein: 25,
  fat: 15,
  carbs: 30,
  weight: 250,
  cost: 0.0012,
  tokens: 235
}
```

## Что тестировать
1. Отправь фото без подписи - должен определить вес автоматически
2. Отправь фото с подписью "250г" - должен использовать указанный вес
3. Проверь, что ответ приходит в правильном формате
